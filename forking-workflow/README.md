# Forking工作流
* 遠端倉庫有公開的和私人的兩個
* 每個人Fork公開的倉庫成私人的倉庫
* 開發都在私人的倉庫中進行
* 完成後發起Pull Request到公開的倉庫
* 公開倉庫的管理者決定是否合併版本
* 優點是不用給所有人公開倉庫的寫入權限
